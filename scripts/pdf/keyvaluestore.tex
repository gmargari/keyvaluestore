\section*{Key-value Store}

\footnotesize
\begin{verbatim}
syntax: ./build/src/sim -c compactionmanager [options]
 COMPACTION MANAGER
    -c compactionmanager:   "nomerge", "immediate", "geometric", "logarithmic",
                            "rangemerge", "cassandra"
    -m memorysize:          memory size in MB (default: 100)
    -r value:               [geometric c.m.] R parameter (default: 3)
    -p value:               [geometric c.m.] P parameter (default: disabled)
    -b blocksize:           [rangemerge c.m.] block size in MB (default: 256)
    -f flushmem:            [rangemerge c.m.] flush memory size in MB
                            (default: 0, when memory is full flush only the biggest range)
    -l value:               [cassandra c.m.] L parameter (default: 4)

 PUT
    -i insertbytes:         number of bytes to insert in MB (default: 1000)
    -n numkeystoinsert:     number of keys to insert (default: 953250)
    -k keysize:             size of keys, in bytes (default: 100)
    -v valuesize:           size of values, in bytes (default: 1000)
    -u:                     create unique keys (default: false)
    -z:                     create zipfian keys (default: false, uniform keys)
    -s:                     read key-values from stdin

 GET
    -g numgetthreads:       number of get threads (default: 0)

 VARIOUS
    -e:                     print key-values that would be inserted and exit
    -o statsperiod:         every that many MB inserted print stats (default: memorysize/2)
    -h:                     print this help message and exit
\end{verbatim}

In all experiments below we insert in a store of \textbf{128MB} of available
memory \textbf{10GB} of data, in order to simulate a datastore of 1GB memory
to which we insert 100GB. Data is inserted as key-value pairs where key is
100B and value is 1KB.
